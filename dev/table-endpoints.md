# Tainacan API Endpoints Table 

This table details the Tainacan API endpoints, their HTTP methods, descriptions, and examples of how to interact with them. 

## 1. Background Processes 

Manages background processes. 

| Endpoint | HTTP Method | Description | Request Parameters (Body/Query) | Execution Example | 
| :------- | :---------- | :-------- | :------------------------------------ | :------------------ | 
| `/bg-processes` | `GET` | Returns a list of background processes. | `user_id` (integer, optional): ID of the processes' owner. Defaults to the current user.<br/>`all_users` (boolean, optional): If `true`, returns processes from all users (admin only). Default `false`.<br/>`status` (string, optional): Filters by status: "open" (running), "closed" (finished/aborted), "all" (all). Default "all".<br/>`perpage` (integer, optional): Number of processes per page. Default `10`.<br/>`paged` (integer, optional): Page to retrieve. Default `1`.<br/>`recent` (boolean, optional): If `true`, returns only recently created or updated processes. Default `false`.<br/>`search` (string, optional): Search term to filter by process type.<br/>`datequery` (array, optional): Array to filter by a `processed_last` date range. E.g., `[{ "after": "YYYY-MM-DD", "before": "YYYY-MM-DD" }]`| ```bash<br/>curl -X GET "[https://example.com/wp-json/tainacan/v2/bg-processes?status=open&perpage=5](https://example.com/wp-json/tainacan/v2/bg-processes?status=open&perpage=5)"<br/>```<br/><br/>```bash<br/>curl -X GET "[https://example.com/wp-json/tainacan/v2/bg-processes?user_id=123&recent=true](https://example.com/wp-json/tainacan/v2/bg-processes?user_id=123&recent=true)"<br/>``` | 
| `/bg-processes/(?P<id>[0-9]+)` | `GET` | Returns a specific background process. | None | ```bash<br/>curl -X GET "[https://example.com/wp-json/tainacan/v2/bg-processes/123](https://example.com/wp-json/tainacan/v2/bg-processes/123)"<br/>``` | 
| `/bg-processes/(?P<id>[0-9]+)` | `POST` | Updates the status of a background process. | `status` (string, **required**): The new process status: "open", "closed", "waiting", "running", "paused", "cancelled", "errored", "finished", "finished-errors".| ```bash<br/>curl -X POST "[https://example.com/wp-json/tainacan/v2/bg-processes/123](https://example.com/wp-json/tainacan/v2/bg-processes/123)" \<br/>     -H "Content-Type: application/json" \<br/>     -d '{"status": "closed"}'<br/>``` | 
| `/bg-processes/(?P<id>[0-9]+)` | `DELETE` | Deletes a background process. | None | ```bash<br/>curl -X DELETE "[https://example.com/wp-json/tainacan/v2/bg-processes/123](https://example.com/wp-json/tainacan/v2/bg-processes/123)"<br/>``` | 
| `/bg-processes/file` | `GET` | Returns the content of a background process log file. | `guid` (string, **required**): The log filename.| ```bash<br/>curl -X GET "[https://example.com/wp-json/tainacan/v2/bg-processes/file?guid=bg-export-123.log](https://example.com/wp-json/tainacan/v2/bg-processes/file?guid=bg-export-123.log)"<br/>``` |
